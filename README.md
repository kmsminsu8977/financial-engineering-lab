# Financial Engineering Portfolio Hub

금융공학, 리스크 모델링, 퀀트 리서치, 금융 데이터 분석 프로젝트를 구조적으로 정리하기 위한 포트폴리오 허브입니다.

이 저장소는 개별 프로젝트의 전체 소스코드를 직접 보관하는 공간이 아니라, 연구 방향과 프로젝트 카테고리, 외부 저장소 및 연구 기록을 연결하는 중심 인덱스 역할을 수행합니다.

## Overview

- 포트폴리오 목적: 금융공학 및 금융 데이터 기반 역량을 일관된 구조로 소개
- 운영 방식: Portfolio Hub -> Project Repository -> Notion Detail -> Slides
- 현재 단계: 공개보다 구조 정비를 우선하는 Phase 1

## Portfolio Categories

### 1. Financial Risk Modeling

금융시장 리스크 모델, 자산가격 모형, 금리 예측, VaR, 시뮬레이션 연구를 다룹니다.

- Category Page: [financial-risk-modeling.md](projects/financial-risk-modeling.md)

### 2. Derivatives and Risk Management

옵션 pricing, 파생상품 구조 분석, 증거금 계산 구조, 리스크 관리 프레임워크를 다룹니다.

- Status: Planned

### 3. Quant Trading Strategy Research

포트폴리오 최적화, 백테스팅, 기술적 지표 기반 전략, 시장 타이밍 연구를 다룹니다.

- Status: Planned

### 4. Financial Data Science

금융 뉴스 NLP, 시장 데이터 분석, 데이터 파이프라인, 시각화, 데이터 마이닝 연구를 다룹니다.

- Status: Planned

## Documentation Principles

- 과장 없이 실제 수행 내용 중심으로 작성합니다.
- 문서 디자인보다 구조와 탐색 가능성을 우선합니다.
- 개별 프로젝트는 향후 별도 repository로 분리합니다.
- 이 저장소에서는 카테고리 설명, 링크 정리, 발표 자료 연결을 담당합니다.

## Current Focus

현재 가장 중요한 목표는 많은 자료를 한 번에 공개하는 것이 아니라, 기존 자료를 재검토하고 재구성 가능한 프로젝트 단위로 정리할 수 있는 기반을 만드는 것입니다.

다음 단계는 카테고리별 프로젝트 후보를 선별하고, 그중 우선순위가 높은 프로젝트부터 개별 repository로 분리하는 것입니다.
